1. Модифицируйте класс MyOpFactory из предыдущей лабораторной так, чтобы он
не создавал экземпляр класса, выполняющего операцию, при каждом вызове
getOpInstance. Обработчики каждой операции должны создаваться по мере
необходимости и помещаться в контейнер. Если экземпляр обработчика
какой-либо операции уже был создан, он должен извлекаться из контейнера.
В качестве контейнера используйте HashMap.

2. Модифицируйте класс Calculator так, чтобы он позволял вводить аргументы
и код операции в формате
<значение a> <код операции> <значение b>
аргументы вычитываются из стандартного потока ввода.
Аргументы отделяются от кода операции пробелами. Для разбора введенной
строки используйте StringTokenizer.

3. Модифицируйте класс Calculator так, чтобы он мог использовать в качестве
входящего и исходящего потоков данных вместо консоли 2 файла( на вход и на выход
соответственно). Причем для вывода результата выполнения операции использовать формат
полученный из файла в формате xml-properties. Для форматирования сообщения можно
воспользоваться классом MessageFormat или методом print класса String.
